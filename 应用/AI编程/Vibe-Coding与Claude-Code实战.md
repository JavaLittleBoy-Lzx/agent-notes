# Vibe Coding 与 Claude Code 实战

> [!abstract] 一句话回答
> **AI 是放大器，不是发动机** — 有方向它放大 10 倍，没方向它放大的是 0。Vibe Coding 时代，程序员的核心任务从"写代码"变成"做决策"。

---

## 💡 核心观点

> 程序员从「我写代码」→ 「我指挥 AI 写代码」

AI 是**放大器**，不是**发动机**：
- 你有方向 → AI 放大 10 倍
- 你没方向 → AI 放大的是 0（甚至负效果）

---

## 🚀 Vibe Coding 是什么？

一种新的编程范式：**你描述目标和意图，AI 生成代码**，你负责审核和决策。

| 传统编程 | Vibe Coding |
|----------|-------------|
| 人写代码 | 人描述，AI 生成 |
| 人调试 | AI 调试，人监督 |
| 线性执行 | 对话式迭代 |

---

## 🔧 Claude Code 能做什么？

基于 Claude Code 一周构建简版 Dify（Hify）的案例，涉及技术：
- 模型管理
- Agent 配置
- 对话引擎
- RAG 知识库
- 工作流编排
- MCP 工具接入

---

## 📚 相关技术栈

| 技术 | 作用 |
|------|------|
| Claude Code | AI 编程工具 |
| Dify | 参考平台（开源 AI Agent 平台）|
| MCP | 工具接入协议 |
| RAG | 知识库增强 |

## 🎯 面试考察点

1. **Vibe Coding 核心思想** — 人做决策，AI 执行
2. **AI 的定位** — 放大器，不是发动机
3. **Claude Code 适用场景** — 快速原型、企业级应用开发
