# Agent 推理模式：ReAct 与 CoT

> [!abstract] 一句话回答
> **CoT** = 让 LLM 先写推理步骤再给答案（纯文字）
> **ReAct** = CoT + 工具调用，交替输出 Thought → Action → Observation 循环
> **ReAct 是目前 Agent 用得最广的模式**，因为它推理过程可见、又能动态利用外部工具

---

## 🎯 面试真题（快手）

> 「 Agent 推理模式有哪些？ReAct 是啥？具体是怎么实现的？」

---

## 🧠 核心问题：LLM 推理的根本困境

LLM 生成答案是 **一个 token 一个 token 地往后预测**。

### 单步问题 → 直接答

```
问：苹果的英文是什么？
答：Apple ✅
```

### 多步问题 → 容易出错

```
问：A公司市值是B的1.2倍，B比C高30%，A和C谁更高差多少？
答：可能直接给一个"感觉对"的答案，但中间推导步骤是隐式的，误差会累积
```

### 类比：心算 vs 笔算

| | 心算 | 笔算 |
|---|---|---|
| 过程 | 在脑子里推导 | 每一步写在纸上 |
| 准确率 | 低，容易跳步 | 高，每步有据可查 |
| 原因 | 中间步骤靠脑子记忆 | 写下来=前文，不会丢 |

**LLM 也一样** — 把推理过程显式写出来，下一步预测就建立在一个清晰的正确基础上。

---

## 三种推理模式

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| **直接输出** | 直接给答案，无中间推理 | 简单问题 |
| **CoT** | 先写推理步骤再给答案 | 复杂推理题 |
| **ReAct** | CoT + 工具调用，推理+行动交替 | 需要外部知识的任务 |

---

## CoT（Chain of Thought）— 思维链

### 核心方法

在 prompt 末尾加一句：**「让我们一步步思考」**

LLM 就会先把推理步骤写出来，再给答案。

### 为什么有效？

LLM 的输出是**顺序生成**的：
1. 先写出来的推理内容 → 进入上下文
2. 下一步的预测 → 建立在这个"已写清的推理"之上
3. 而不是靠脑子在"暗中维持"中间状态

就像笔算，纸上的每一行数字都在帮你记住上一步算到哪了。

### 两种触发方式

| 方式 | 怎么做 | 效果 |
|------|--------|------|
| **Zero-shot CoT** | prompt 末尾加「让我们一步步思考」 | 不需要示例，LLM 自己展开 |
| **Few-shot CoT** | 给几个完整推理例子让 LLM 模仿 | 更稳定，适合固定格式场景 |

### CoT 的致命局限

> **它是纯文字推理，无法和外部世界交互。**

```
问：现在苹果公司的市值是多少？
CoT 只能答：基于训练数据，大概是xxx（可能已过时）
```

推理再完整，也拿不到**实时数据**，这就是 CoT 的天花板。

---

## ReAct（Reasoning + Acting）

### 核心思路

在 CoT 的推理链里**插入真实的「行动」**。

让 LLM 按 **Thought → Action → Observation** 循环推进：

```
Thought（思考）→ Action（调用工具）→ Observation（获取结果）→ 重复直到 → Final Answer
```

### 完整实例：查苹果和谷歌市值

> 问：2024年苹果和谷歌谁市值更高？差多少？

**CoT 只能猜**（知识可能过时），**ReAct 真的去查：**

```
Thought: 这道题需要两家公司的实时市值数据，我得先查苹果
Action: search
Action Input: 苹果公司 2024 年市值
Observation: 苹果公司 2024 年市值约为 3.5 万亿美元

Thought: 好，苹果的数字有了，再查谷歌的
Action: search
Action Input: 谷歌 2024 年市值
Observation: 谷歌 2024 年市值约为 2.1 万亿美元

Thought: 两个数字都有了，苹果 3.5 万亿，谷歌 2.1 万亿，差距 1.4 万亿
Final Answer: 苹果市值约 3.5 万亿美元，谷歌约 2.1 万亿美元，苹果更高，差距约 1.4 万亿美元
```

---

## ReAct 实现原理

> [!warning] 重要澄清
> **ReAct 的循环不是 LLM 自己转，而是代码驱动的。**

### 代码驱动流程

```
LLM 输出 Thought + Action
        ↓
代码检测：有没有 Final Answer？
        ↓ 无
解析 Action，执行工具，把结果填入 Observation
        ↓
再次调用 LLM，传入更新后的历史
        ↓
循环，直到 LLM 输出 Final Answer
```

### 标准 ReAct prompt 格式

```
你是一个 AI 助手，可以使用以下工具：
- search(query): 搜索互联网获取最新信息
- calculator(expr): 计算数学表达式

回答格式：
Thought: 你的思考过程
Action: 工具名称
Action Input: 工具输入参数
Observation: （此行由系统填入，你不用写）
... 可重复多轮 ...
Final Answer: 最终答案
```

### 两种实现方式

| 方式 | 描述 |
|------|------|
| **Plan-and-Execute** | 先让 LLM 输出完整计划，再按计划逐步执行 |
| **Error-Recovery** | 边执行边规划，根据每步结果动态调整 |

---

## CoT vs ReAct 对比

| | CoT | ReAct |
|---|---|---|
| 推理过程可见 | ✅ | ✅ |
| 外部工具调用 | ❌ | ✅ |
| 实时数据获取 | ❌ | ✅ |
| 适用场景 | 纯逻辑推理 | 需要实时信息/执行操作 |

---

## 🎯 面试考察点

1. **三种推理模式的区别** — 直接输出 / CoT / ReAct
2. **CoT 为什么有效** — 把隐式推导变成显式上下文
3. **CoT 的局限** — 纯文字，无法获取实时数据
4. **ReAct 的循环** — Thought → Action → Observation → 循环
5. **谁在驱动循环** — 代码驱动，不是 LLM 自己转
6. **ReAct vs CoT 的选择** — 是否需要外部数据
