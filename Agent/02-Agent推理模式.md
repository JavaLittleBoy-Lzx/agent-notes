# Agent 推理模式：ReAct 与 CoT

## 面试题

快手面试题真题：「Agent 推理模式有哪些？ReAct 是啥？具体是怎么实现的？」

---

## 一句话回答

Agent 的推理模式主要有：

- **直接输出**：无中间推理，直接给答案
- **CoT（思维链）**：让 LLM 先写出推理过程再给答案，准确率更高
- **ReAct**：在 CoT 基础上加入「行动」，交替输出思考和工具调用，形成「思考→行动→观察」循环

**ReAct 是目前 Agent 用得最广的模式**，因为它推理过程可见、又能动态利用外部工具。

---

## 核心概念解析

### 什么是推理模式？

LLM 的根本困境：多步推导时，误差会在中间某个暗处悄悄累积，最终暴露在答案里。

**类比：**
- 心算「123 × 456」可能算错
- 笔算把每一步写下来，算错概率大大降低

原因：「写下来的过程」本身帮助避免在中间某步跳跃出错。

### CoT（Chain of Thought）

**核心原理：** 在 prompt 里加一句「让我们一步步思考」，LLM 会先把推理步骤写出来，再给答案。

**为什么有效？**
- LLM 输出是顺序生成的
- 先写出来的推理内容会进入上下文，成为后续生成的依据
- 下一步的预测建立在一个已经写清楚的正确基础上

**两种触发方式：**

| 方式 | 原理 | 适用场景 |
|------|------|----------|
| **Zero-shot CoT** | prompt 末尾加「让我们一步步思考」 | 不需要额外示例 |
| **Few-shot CoT** | 给几个带完整推理过程的例子让 LLM 模仿 | 格式要求固定、效果更稳定 |

**CoT 的局限：** 纯文字推理，无法和外部世界交互，拿不到实时数据。

### ReAct（Reasoning + Acting）

**核心思路：** 在 CoT 的推理链里插入真实的「行动」，让 LLM 按「**思考 → 行动 → 观察**」循环推进。

**循环流程：**
```
Thought: 思考当前该怎么做
   ↓
Action: 调用工具获取信息或执行操作
   ↓
Observation: 工具返回结果作为新的观察
   ↓
（重复直到）
   ↓
Final Answer: LLM 判断任务完成
```

**ReAct 实例演示：**

> 问题：「2024 年苹果公司和谷歌的市值谁更高？差多少？」

```
Thought: 这道题需要两家公司的实时市值数据，我得先查苹果的市值
Action: search
Action Input: 苹果公司 2024 年市值
Observation: 苹果公司 2024 年市值约为 3.5 万亿美元

Thought: 好，苹果的数字有了，再查谷歌的
Action: search
Action Input: 谷歌 2024 年市值
Observation: 谷歌 2024 年市值约为 2.1 万亿美元

Thought: 两个数字都有了，苹果 3.5 万亿，谷歌 2.1 万亿，苹果更高，差距是 1.4 万亿
Final Answer: 苹果公司市值约 3.5 万亿美元，谷歌约 2.1 万亿美元，苹果更高，差距约 1.4 万亿美元
```

### ReAct vs CoT

| | CoT | ReAct |
|---|---|---|
| 推理过程 | ✅ 可见 | ✅ 可见 |
| 外部工具调用 | ❌ 不支持 | ✅ 支持 |
| 实时数据获取 | ❌ 不支持 | ✅ 支持 |
| 适用场景 | 纯逻辑推理 | 需要实时信息/执行操作 |

---

## ReAct 实现原理

**ReAct 的循环不是 LLM 自己在转，而是由代码驱动的：**

```
LLM 输出 Thought + Action
       ↓
代码检测输出，判断「有没有 Final Answer」
       ↓
若无 → 解析 Action，执行工具
       ↓
把工具结果填入 Observation
       ↓
再次调用 LLM
       ↓
（循环）
```

**典型的 ReAct prompt 格式：**
```
你是一个 AI 助手，可以使用以下工具：
- search(query): 搜索互联网获取最新信息
- calculator(expr): 计算数学表达式

回答时请严格按照以下格式：
Thought: 你的思考过程（分析当前情况，决定下一步）
Action: 工具名称
Action Input: 工具的输入参数
Observation: （此行由系统填入工具返回的结果，你不用写）
... 以上可以重复多轮 ...
Final Answer: 当你确定可以回答时，给出最终答案
```
