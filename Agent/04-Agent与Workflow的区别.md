# Agent 与 Workflow 的区别

> [!abstract] 一句话回答
> **Agent** = 自主决策，动态选择下一步；**Workflow** = 预设流程，固定路径执行。

---

## 🔄 核心区别

| | Agent | Workflow |
|---|---|---|
| **决策方式** | LLM 自主判断 | 预设节点，固定顺序 |
| **执行路径** | 动态，根据结果调整 | 固定流程图 |
| **灵活性** | 高，可应对未知场景 | 低，适合固定业务流程 |
| **适用场景** | 复杂推理、多工具调用 | 数据处理流水线、审批流 |

---

## 🧠 Agent 的特点

- **自主决策**：下一步做什么，由 LLM 根据当前状态判断
- **动态工具选择**：可以决定调用哪个工具，或者不调用工具直接给答案
- **适应性**：遇到意外情况可以调整策略

```
用户问题
    ↓
LLM 判断：该搜索？该计算？该直接回答？
    ↓
→ 搜索 → 拿到结果 → LLM 再判断
→ 计算 → 拿到结果 → LLM 再判断
→ 直接回答 → 结束
```

---

## ⚙️ Workflow 的特点

- **预设流程**：每一步都是固定的，节点 A → 节点 B → 节点 C
- **确定性**：执行路径完全可预测
- **适合场景**：数据清洗 → 格式转换 → 发送报告

```
节点A（数据清洗）
    ↓
节点B（格式转换）
    ↓
节点C（发送邮件）
    ↓
结束
```

---

## 🎯 什么时候用哪个？

| 场景 | 推荐 |
|------|------|
| 客服问答，需要理解意图再行动 | **Agent** |
| 每日自动报表生成 | **Workflow** |
| 复杂研究任务，不知道要走几步 | **Agent** |
| 固定审批流程 | **Workflow** |

---

## 🎯 面试考察点

1. **说出核心区别** — 自主决策 vs 预设流程
2. **举实际例子** — 客服用 Agent，审批流用 Workflow
