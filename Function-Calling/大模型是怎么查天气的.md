# 大模型是怎么查天气的？

> [!abstract] 一句话回答
> LLM 通过 **Function Calling** 机制，输出结构化的调用指令（如 `{tool: "get_weather", args: {city: "北京"}}`），由程序真正执行 API 调用，结果再传回 LLM 生成回答。**LLM 负责决定"做什么"，程序负责"真正执行"**。

---

## 🔄 Function Calling 工作流程

```
用户：今天北京天气怎么样？
        ↓
LLM 分析意图：需要调用天气API
        ↓
LLM 输出：{tool_name: "get_weather", arguments: {city: "北京"}}
        ↓ ← 程序在这里介入
程序执行：调用天气API，获取真实数据
        ↓
结果反馈给 LLM：「北京今天晴，25度」
        ↓
LLM 生成最终回答
```

---

## 🛠️ 工具定义示例（OpenAI 格式）

```json
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "获取指定城市的天气信息",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {
          "type": "string",
          "description": "城市名称"
        },
        "unit": {
          "type": "string",
          "enum": ["celsius", "fahrenheit"]
        }
      },
      "required": ["city"]
    }
  }
}
```

---

## ⚠️ 关键澄清

> LLM 本身**不会真的发 HTTP 请求**，它只是输出 JSON 格式的调用指令。
> 实际的网络请求是由**你的代码**完成的。

---

## 🎯 面试考察点

1. **说出 Function Calling 的流程** — LLM 返回 JSON → 程序执行 → 结果反馈 LLM
2. **核心分工** — 模型负责决策，程序负责执行
3. **工具定义格式** — name / description / parameters / required
